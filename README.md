# sag.gaming

Personal knowledge base for games I actively play. Guides, references, mechanics notes and build documentation — built to retain information across sessions and time.

Content is written from personal experience and verified against primary sources where possible. Unverified information is explicitly marked.

---

## Structure

Each game folder follows the same structure:

```
game-name/
├── README.md        — overview, version, quick links
├── guide.md         — main practical guide
├── pending.md       — unverified information to check in-game
└── changelog.md     — patch notes relevant to the guide
```
